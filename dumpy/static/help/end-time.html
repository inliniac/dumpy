The end time for packets to return. Only events occuring at or before this time will be returned.

Absolute timestamp (ISO 8601 format) or relative time strings can be used.

Absolute Timestamps

Absolute timestamps are based around ISO 8601 formatting. If no timezone offset is provided the offset of the browser will be used (currently GMT{offset}). Only the resolution required needs to be specified with the lower resolution being a microsecond.

Input Result
2012 2012-01-01T00:00:00{offset}
2012-02 2012-02-01T00:00:00{offset}
2012-02-03T11 2012-02-03T11:00:00{offset}
2012-02-03T11:59:01.123456+1200 2012-02-03T11:59:01.123456+1200
2012-02-03 11:59:00Z 2012-02-03T11:59:00Z

Relative Timestamps

A relative timestamp references a time after a reference time. If the provided filter is an event containing a timestamp the reference time will be the time from the event. Otherwise the reference time will be the start time.

Examples:

  • 2 seconds
  • 3 minutes
  • 4 hours
  • 5 days
